⑥ 名詞：「YOLO 腿 / VLM 腿」— 兩條特徵支柱

影像 →YOLO 腿（符號腿）：seg 偵測 → 145 維手工特徵（bbox 交疊、質心速度、1/3/8s 時窗）─┐
影像 →VLM 腿（語意腿）：凍結 ViT → 每秒 patch 向量平均（瞬時、零時窗、零標注）─┴→ concat → MLP → state
  • v3/v4 = 只用 YOLO 腿 · LB/DV2 = 只用 VLM 腿 · FUS = 兩條腿串接
  • 「誰殺了 YOLO 腿」= 打亂殺的是 YOLO 腿的手工時序特徵（時窗吃到剪接點），不是偵測本身 — mask 每幀照樣準；VLM 腿零損傷（bit 同）；FUS 一腿受傷所以跛行不倒（−.080）
  • 嫌行話可改口：「YOLO 特徵路徑」「符號特徵那一路」

⑦ 糊糊色塊（VLM PCA / 融合疊圖）怎麼講

30 秒台詞：「這是 VLM 的視角。它不畫框 — 把畫面切成 14×14 小格，每格編碼成 384 維語意向量；我們用 PCA 壓到 3 維當 RGB 上色。同顏色 = backbone 認為同一種東西： 鍋一坨色、手臂一坨色、檯面一坨色，顏色邊界 = 它看到的物件邊界。 YOLO 說『這裡有個 wok』，VLM 給的是整片畫面的語意場。」
  • 為什麼糊？patch 解析度 14×14（DV2 16×16）放大到 640×360 自然糊 — 糊的是圖，不是特徵：就是這個糊糊特徵在打亂測試幾乎不掉分（92→89）， 比清楚的 YOLO mask 腿（92→49）穩
  • 圖 ≠ 判斷輸入：圖是 384 維內部特徵的 3 維投影切片（給人看）；模型實際吃每秒 196 個 patch 向量的平均（瞬時 → 打亂免疫的根源）
  • FUS 格是疊圖隱喻，非模型輸出：v4 mask（65%）疊 LB 色場（35%）= 符號腿 + 語意腿 同框；實作 = 兩種特徵 concat 進同一顆分類器
  • Demo 殺手鐧：指出色塊跟著物件走 — 飯下鍋、蒸氣起來時鍋區顏色改變 = 零標注下它在追蹤「鍋子狀態」，state 資訊就從這來
追問answer
顏色本身有意義嗎？相對有：同 run 內同色 = 同類（PCA 每 run 擬合一次，顏色時間一致、跟著鍋子走）；跨 run 不可比
LB 跟 DV2 配色為何不同？不同 backbone 語意空間不同 → PCA 軸不同；但都分得出鍋/臂/檯面 = 零標注也有結構的證據
這是 attention map 嗎？不是。attention = 模型在看哪；這是 token 內容的投影 =「它把畫面分成什麼」，更直接

⑧ 講稿三句話 ＋ Q&A 防身

1. 原序部署：融合最強 — FUS 94.0%（因果 93.0%），全系統遠勝規則 FSM 0.699。
2. 打亂 = 順序魯棒性壓力測試 — 手工時序特徵腿隨切碎密度崩（v4: 92→80→86→73→49）， 凍結 VLM/DINOv2 embedding 是瞬時特徵，幾乎不動（92→89）：順序魯棒性住在 embedding 腿。
3. 不是 model 壞，是特徵設計的取捨 — 時序窗換來原序精度，代價是剪接脆弱； 若要順序魯棒可加 splice augmentation 重訓融合（管線現成）。
  • 「作弊嗎？」→ 預設 m2v 是離線上限（chunk 內看未來，+2~5pt）；因果 m2 頁面全數字都在，結論不變
  • 「LB/DV2 為何不掉？」→ 每秒 emission 打亂前後 bit 級相同（實測），只剩 HMM 先驗小額罰款
  • 「準確率怎麼算？」→ 每秒 label 全等 / 總秒數，LODO，GT 來自 /play_recipe feedback
  • 「30s 打散為何反而變好？」→ 切點對齊 30s chunk，chunk 內容大多完整連續
  • 工具：四頁 player（原序/打亂 × 離線/因果）+ 六格 model internals（YOLO mask / ViT PCA）同步影片
